## 3. Accessibility Audit (WCAG 2.2 AA)

### Perceivable

- [x] All images have meaningful `alt` text
- [x] Decorative images use `alt=""` or `aria-hidden="true"`
- [x] Color contrast — All text meets 4.5:1 minimum (most exceed 7:1)
- [x] Information not conveyed by color alone (icons + text + color used together)
- [x] `aria-label` on icon-only buttons (hamburger, close, social icons)
- [x] `aria-label` on landmark elements (navigation, main content)

### Operable

- [x] Skip-to-content link — First focusable element
- [x] All interactive elements keyboard-accessible
- [x] Visible focus indicators (`:focus-visible` with cyan outline)
- [x] No keyboard traps (mobile menu closes with Escape)
- [x] Touch targets minimum 44×44px
- [x] Smooth scrolling (`scroll-behavior: smooth`)
- [x] `prefers-reduced-motion` respected (all animations disabled)

### Understandable

- [x] Semantic HTML — `<nav>`, `<main>`, `<footer>`, `<section>`, `<article>`
- [x] Proper heading hierarchy (h1 → h2 → h3, no skipping)
- [x] Form fields have `<label>` elements
- [x] Form errors use `role="alert"` for screen readers
- [x] `aria-expanded` on accordion and mobile menu toggles
- [x] `aria-current="page"` for active navigation
- [x] `aria-haspopup="true"` on mega-menu

### Robust

- [x] ARIA landmarks — `banner`, `navigation`, `main`, `contentinfo`
- [x] `lang="en"` on `<html>` element
- [x] Valid HTML structure
- [x] `role="region"` with `aria-labelledby` on expandable content

### Recommendations

- [ ] Add `aria-current="page"` to active nav links
- [ ] Add focus trapping in mobile menu and modals
- [ ] Test with actual screen readers (NVDA, VoiceOver)

## 4. Security Audit

### Implemented

- [x] Content Security Policy (CSP) — Restricts scripts, styles, fonts, connections
- [x] Strict-Transport-Security — HSTS with 2-year max-age, preload
- [x] X-Content-Type-Options — `nosniff`
- [x] X-Frame-Options — `DENY` (prevents clickjacking)
- [x] Referrer-Policy — `strict-origin-when-cross-origin`
- [x] Permissions-Policy — Restricts camera, mic, geolocation
- [x] X-XSS-Protection — Legacy protection for older browsers
- [x] `poweredByHeader: false` — Hides Next.js framework info
- [x] Honeypot field in forms (spam protection)
- [x] Form submission via third-party services (no backend to exploit)
- [x] All external links use `rel="noopener noreferrer"`
- [x] `form-action 'self'` in CSP restricts form submissions
- [x] Environment variables for all API keys and service IDs

### Recommendations

- [ ] Add `security.txt` file at `/.well-known/security.txt`
- [ ] Implement rate limiting via Vercel WAF (future)
- [ ] Add `Cross-Origin-Embedder-Policy` header
- [ ] Regular dependency audits (`npm audit`)
- [ ] Add reCAPTCHA for form submissions (if spam becomes an issue)
